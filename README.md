# 🥋 TeamPro 教練共同備課系統

手機優先的道館教練共同備課系統。從**學員名單統計 → 教練共同備課 → 課程分組 → 整週計畫 → 繁中 PDF 輸出**，一條龍完成。

不是點名系統，也不是學校行政系統，而是**教練群一起把每天 90 分鐘的課排好、討論好、印出來**的工具。

---

## 一、技術架構（為什麼是單檔 HTML）

| 項目 | 採用 |
|------|------|
| 前端 | 原生 HTML + CSS + JavaScript（單檔 `index.html`，零建置） |
| 資料庫 | **IndexedDB**（瀏覽器內建的持久化資料庫，非記憶體、重整不消失） |
| Excel 解析 | SheetJS（`xlsx`，CDN 載入） |
| PDF | **瀏覽器原生列印**（`window.print()` + `@media print` A4 版面）→ 中文由系統字型渲染，**零亂碼**，可直接「另存為 PDF」 |
| 部署 | 直接雙擊開啟，或上傳 GitHub Pages / 任意靜態空間 |

**為何不用規格書指定的 Next.js + Supabase？**
本專案定位是「單一道館、教練群共用一台平板／手機」的現場工具，需要**免帳號、免後端、雙擊即用、離線可用**。IndexedDB 已滿足「資料持久化、重整不消失」的驗收要求。若日後需要**多裝置雲端即時共用**，可外接 GAS+Sheets 或 Firebase 作為同步層（見「後續優化」），前端邏輯不需重寫。

> 唯一的外部相依是 SheetJS 的 CDN（僅 Excel 匯入／範本下載時使用）。若需完全離線，可將 `xlsx.full.min.js` 下載到本機並改成相對路徑引用。

## 二、本機執行

1. 直接用 Chrome / Edge / Safari 開啟 `index.html`（雙擊即可）。
2. 首次使用：登入頁點 **「🎁 載入示範資料」** → 建立 40 位學員、8 個帳號、3 個班別、一週示範課程。
3. 用下方任一帳號登入（PIN 皆為 `1234`）。

> 建議放在手機加入主畫面，或用平板當道館共用機。

## 三、部署到 GitHub Pages

```bash
cd D:\TeamPro共同備課系統
git init
git add .
git commit -m "TeamPro 共同備課系統"
git branch -M main
git remote add origin https://github.com/<你的帳號>/teampro-lesson-planner.git
git push -u origin main
```
到 GitHub repo → Settings → Pages → Source 選 `main` / `root` → 儲存。
數分鐘後即可用 `https://<你的帳號>.github.io/teampro-lesson-planner/` 開啟。

> 單機模式下，每台裝置資料各自存在該裝置瀏覽器的 IndexedDB。要**多位教練各自手機共用同一份資料**，請做下面的雲端同步。

## 三之二、多裝置雲端同步（各教練各自手機用）

用 Google Apps Script + 試算表當同步後端，免費、資料在你的 Google 帳號。

1. 開一個新的 Google 試算表 → **擴充功能 → Apps Script**。
2. 把 `apps-script/Code.gs` 整段貼進去，儲存。
3. **部署 → 新增部署 → 網頁應用程式**；執行身分＝我，存取權限＝**任何人** → 部署 → 複製 `/exec` 網址。
4. 把該網址填進 `index.html` 最上面的 `const DEFAULT_SYNC_URL='...'`，再上架 GitHub Pages。
   （或先登入 → **設定 → 多裝置雲端同步** 貼上網址測試。但要多人共用，務必寫進 `DEFAULT_SYNC_URL` 再上架。）
5. **主帳號（楊復傑教練）**先在一台裝置登入 → 載入／整理好資料 → 設定頁按 **⬆️ 上傳本機到雲端**。
6. 其他教練打開**同一個網址** → 自動把雲端資料拉下來 → 用自己的帳號登入即可。之後任何人存檔會即時上傳，其他人每 15 秒自動更新（切換頁面也會更新）。

**同步規則**：以「最後修改時間」為準（Last-Write-Wins）。同一筆資料兩人同時改，較晚存的會覆蓋較早的——小團隊備課足夠，但要避免兩人同時編輯同一天同一組。

## 五、主要功能對照

1. **Excel / CSV 匯入**：下載範本 → 選檔 → 預覽（新增/更新/錯誤筆數，錯誤列標示原因）→ 確認匯入。年齡層只收「幼兒/國小/國中/成人」，上課日支援「一、三、五 / 星期一… / 週一…」自動標準化。
2. **今日人數統計**：依日期＋班別即時算總數、四年齡層、各級別段位、各專項、準備晉級、比賽選手、可出席教練；卡片可點開看名單；學員頁多條件篩選＋搜尋。
3. **系統分組建議**：9 種模式（智慧綜合／年齡／級別／品勢對練／初中高階／幼兒獨立／成人獨立／晉級加強／比賽選手），會說明分組原因與教練人力提醒；可點選/拖曳搬移學員、增刪併拆組、指派主教練與助教；自動檢查未分組、重複分組、無教練、超額、同教練撞組。
4. **90 分鐘五階段備課**：0-10 做操慢跑（一鍵套用）／10-30 熱身體能（能力標籤）／30-60 分組訓練（各組獨立填基本動作·品勢·腿法·對練·目標·常見錯誤）／60-85 應用挑戰（銜接提示）／85-90 講評。每階段可標記完成。
5. **教練共同備課討論室**：每日獨立討論、回覆、標時段、時間、已讀數、編輯保留紀錄。
6. **課程狀態流**：草稿→討論中→待確認→已發布→已完成→已鎖定（發布/鎖定含權限與稽核）。
7. **整週計畫**：週一至週日檢視、每日人數與年齡/級別分布、備課完成度、複製上一堂/複製上週/複製到其他班；本週能力分析提醒（敏捷/核心/爆發力/品勢/對練次數、缺柔軟平衡協調提醒，僅提醒不擋發布）。
8. **我的今日任務（助教）**：只看自己負責組別的名單、內容、目標、器材、注意事項與主教練留言。
9. **課後紀錄**：實到人數、完成度、原因、反應、表現好/需補強、需調整分組、**下堂課銜接內容（自動帶到下次備課提示）**、安全事件、傷勢。
10. **PDF 輸出**：完整管理版／教練現場執行版，A4 直式，中文正常，可列印可存 PDF，檔名 `道館_週課程計畫_起~迄`。
11. **系統設定**：道館名稱、Logo、各專項分組人數上限、能力分組級別、JSON 備份匯出/匯入、稽核紀錄。

## 八、已知限制

- **同步為選用**：預設單機（資料存各裝置瀏覽器）。要多裝置即時共用需依「三之二」部署 GAS 後端；同步採 Last-Write-Wins，非逐欄位合併。
- **清除瀏覽器資料**會一併清掉 IndexedDB，請定期匯出備份。
- Excel 匯入需網路載入 SheetJS CDN（可改本地檔離線化）。
- 拖曳分組在桌機用原生拖放，手機建議用「點學員→點目標組」搬移。
- 討論的「@教練」目前為純文字標註，未做通知推播。

## 十一、教練操作手順（給第一次使用的教練）

系統分成 5 大區（手機底部：**今日 / 本週 / 我的任務 / 課程庫 / 更多**）。

### 課前備課（主教練，約 3 分鐘）
從「**今日**」按最上面的主按鈕（**建立今日課程 / 繼續完成今日備課**），依 7 步走：
1. **課程資料** — 選今天的班別（系統已帶出）。
2. **確認人數** — 逐人到／假／未到，或「一鍵全到」、「＋臨時到課」；下方看年齡／程度／專項統計。
3. **訓練分組** — 按「產生分組建議」，可拖曳/點選搬移學員、指派教練。
4. **課程內容** — 五階段摺疊卡；10–30 分點「＋熱身快選」挑項目、湊滿 20 分。
5. **教練任務** — 檢查每組的教練、學員、內容、器材、注意。
6. **確認發布** — 看發布前檢查，按「發布今日課程」。
7. **課後紀錄** — 上完課後填（見下）。
> 想更快：今日卡片的「**快速備課**」可一鍵產生初稿。

### 上課現場（所有教練）
今日卡片按「**開始上課**／查看今日執行內容」進入**上課模式**：大字顯示目前階段、倒數、各組重點、負責教練、器材、安全提醒；大按鈕可「完成本階段 / 提前下一階段 / 延長 1・5 分 / 查看分組 / 紀錄傷勢」。現場**不會**出現任何管理功能。

### 課後回報（60 秒）
「今日」卡片變成「**完成課後回報**」或從發布步驟接續：填實到人數、完成度、需補強學生、**下堂課接續內容**（會自動出現在下一次同班備課提醒）。

### 助教
登入直接進「**我的任務**」：今天負責的組別、學員名單、教學內容、器材、注意、主教練留言，並可快速回報。助教看不到管理功能。

### 哪裡建立 / 修改資料
- **更多（管理中心）** → 學員 / 班別 / 教練 / 系統設定 / 同步備份（僅管理者，學員含主教練）。
- 每頁右上「**＋新增**」、手機右下**浮動＋鈕**；每筆資料一致提供 **編輯 / 複製 / 停用**。
- 課程模板、熱身動作庫、歷史課程、PDF 都在「**課程庫**」。

### 首次建置
全新系統登入頁按「**開始設定精靈**」：道館 → 選雄麒班別 → 建立管理者與教練 → 學員，完成後直接進今日工作台。
